# Reglas de documentación — OBLIGATORIO

Al finalizar cada sesión de trabajo (o cuando el usuario pregunte qué queda pendiente):

1. **Actualizar PLAN.md**: marcar con [x] todo lo completado, agregar nuevos módulos/tareas si se crearon, actualizar la fecha y el bloque ESTADO ACTUAL.
2. **Actualizar ARQUITECTURA.md**: si se tomó una decisión técnica relevante (nueva tabla, nuevo servicio, nuevo patrón), documentarla.
3. **No listar como pendiente algo que ya se hizo** — leer PLAN.md antes de responder qué falta.

El objetivo es que la próxima sesión arranque con contexto real, sin repetir lo ya hecho.
